# Documentation Policy

CRITICAL RULE: Do NOT create documentation files (.md) or deployment scripts (.sh, .bash) unless explicitly requested by the user.

This includes:
- README files
- QUICKSTART guides
- SUMMARY documents
- DEPLOYMENT checklists
- HOW-TO guides
- TROUBLESHOOTING guides
- Any other .md files

Focus only on:
- Actual code implementation
- Configuration files
- Source code files
- Tests (if requested)

If the user needs documentation, they will explicitly ask for it.
