Linux Commands Cheat Sheet

Quick reference for managing the game server directory at /srv/games/icarus

Navigation
cd /srv/games/icarus

Navigate to the Icarus dedicated server root directory where all game files, configs, and saves are stored.

ls -lah

List all files and folders including hidden ones, with permissions, owner, size in human-readable format, and modification dates.

pwd

Print the full path of your current working directory. Useful when you're not sure where you are in the filesystem.

File Operations
cat filename.txt

Print the entire contents of a file to the terminal. Best for small files — for large files use less instead.

tail -f logfile.log

Follow a log file in real-time, showing new lines as they're written. Great for watching server output live. Press Ctrl+C to stop.

grep "error" logfile.log

Search a file for lines containing the given text. Case-sensitive by default — add -i for case-insensitive search.

Disk Usage
du -sh *

Show the total size of each file and folder in the current directory. Helpful for finding what's eating up disk space.

df -h

Show how much disk space is used and available on each mounted filesystem. Check this if the server is running low on storage.

Process Management
ps aux | grep icarus

List all running processes and filter for Icarus-related ones. Shows PID, CPU/memory usage, and the full command. Use the PID with kill if needed.

top

Live view of system resource usage — CPU, memory, and running processes sorted by activity. Press q to quit, M to sort by memory.

Permissions
ls -l

List files with their permission flags (e.g. rwxr-xr-x), owner, group, and size. The first column shows read/write/execute for owner, group, and others.

chmod +x script.sh

Add execute permission to a script so it can be run directly with ./script.sh. Without this, you'd get a "Permission denied" error.

Server Configuration
nano Icarus/Saved/Config/WindowsServer/ServerSettings.ini

Open the server config file in the nano text editor. Change game settings like difficulty, max players, and session name. Press Ctrl+O to save, Ctrl+X to exit.

nano start.sh

Edit the server startup script that controls how the game server launches, including Wine/Proton settings and launch arguments.

About start.sh:

  • Launches Icarus server using Wine/Proton compatibility layer
  • Manages tmux session for background server operation
  • Checks for existing server processes to prevent duplicates
  • Logs startup information for troubleshooting
Caution: Be careful with commands that modify or delete files. Always double-check before running destructive operations.
